## 🚀 Featured Projects

### ✈️ SkyElite — AI Travel Recommendation System
> 🥉 **3rd Place, National AI Hackathon (atomcamp)** · [GitHub](https://github.com/r-muhammadahmadabbas/ai-travel-planner)

A passport-aware, budget-aware AI travel recommendation system that acts as a **decision assistant**, not a generic travel blog.

<details>
<summary><strong>🔍 Architecture Deep Dive</strong></summary>
<br>

**Pipeline:** 7-node LangGraph `StateGraph` with GCF context optimization
```
intake → filter → visa → research → scoring → tradeoff → final
```

**What makes it special:**
- Filters physically impossible destinations based on passport nationality and visa rules
- Scores destinations across **safety, budget, visa difficulty, hotels, and scenery**
- Returns ranked packages with **confidence levels, source counts, data origin (seed/live/mixed), and honest tradeoffs**
- Runs with **zero API keys** — in-memory persistence, mock email, and deterministic synthesis are defaults
- Adding keys progressively upgrades capabilities (Supabase → Claude → Tavily → Resend)

**Security Architecture:**
- CORS allow-listing, rate limiting, Pydantic v2 input validation
- Prompt-injection sanitization on all web content
- RLS-enforced Supabase persistence with service-role key bypass
- Generic error responses to prevent information leakage

**Design Patterns Used:**
Repository · Abstract Factory · Adapter · Strategy · Facade · Template Method · Null Object

</details>

| Layer | Tech |
|---|---|
| Frontend | Next.js 15, TypeScript, Tailwind CSS, Three.js |
| Backend | FastAPI, Pydantic v2, LangGraph |
| AI | Groq API (LLM synthesis) |
| Database | Supabase (Postgres + RLS) |
| Deployment | Vercel + Railway + Supabase |

### 🧠 Nexus AI — Multi-Agent Research Assistant
> Automated industry research with 4 coordinated AI agents · [GitHub](https://github.com/r-muhammadahmadabbas)

<details>
<summary><strong>🔍 Architecture Deep Dive</strong></summary>
<br>

**Agent Pipeline:**
```
User Query → Clarity Agent → Research Agent → Validator Agent → Synthesis Agent → Streamed Report
```

**Key Engineering Decisions:**
- **4 Specialized Agents:** Clarity (understands intent), Research (gathers data via Tavily), Validator (fact-checks), Synthesis (produces report)
- **Real-time streaming** via FastAPI Server-Sent Events — intermediate agent states + token-by-token Markdown reports
- **Sliding-window memory** to keep context focused across long research sessions
- **Token-aware result truncation** to prevent Groq API rate-limit bottlenecks
- **Human-in-the-Loop** clarification loop when query intent is ambiguous

</details>

| Component | Tech |
|---|---|
| Orchestration | LangGraph, LangChain |
| Backend | FastAPI, Server-Sent Events (SSE) |
| Research | Tavily API |
| LLM | Groq API |
